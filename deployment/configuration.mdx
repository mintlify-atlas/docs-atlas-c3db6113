### Database Configuration

#### H2 In-Memory Database (Default)

The application uses H2 in-memory database for development:

```properties
spring.datasource.url=jdbc:h2:mem:testdb
spring.jpa.defer-datasource-initialization=true
spring.jpa.hibernate.ddl-auto=create-drop
spring.datasource.initialize=true
```

**Properties explained:**

- `spring.datasource.url=jdbc:h2:mem:testdb` - In-memory database named "testdb"
- `spring.jpa.defer-datasource-initialization=true` - Wait for schema creation before data initialization
- `spring.jpa.hibernate.ddl-auto=create-drop` - Create schema on startup, drop on shutdown
- `spring.datasource.initialize=true` - Enable database initialization

<Warning>
  **Data Loss Warning:** With `create-drop` mode, all data is lost when the application stops. This is intended for development only.
</Warning>

#### H2 File-Based Database

For persistent storage during development:

```properties
spring.datasource.url=jdbc:h2:file:./data/crmdb
spring.jpa.hibernate.ddl-auto=update
```

This creates a file-based database in `./data/crmdb.mv.db`

#### PostgreSQL Configuration

For production deployment with PostgreSQL:

<Steps>
  <Step title="Add PostgreSQL dependency to pom.xml">
    ```xml
    <dependency>
      <groupId>org.postgresql</groupId>
      <artifactId>postgresql</artifactId>
      <scope>runtime</scope>
    </dependency>
    ```
  </Step>

  <Step title="Update application.properties">
    ```properties
    spring.datasource.url=jdbc:postgresql://localhost:5432/crmdb
    spring.datasource.username=crmuser
    spring.datasource.password=your_password
    spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect
    spring.jpa.hibernate.ddl-auto=validate
    ```
  </Step>
</Steps>

#### MySQL Configuration

For production deployment with MySQL:

<Steps>
  <Step title="Add MySQL dependency to pom.xml">
    ```xml
    <dependency>
      <groupId>com.mysql</groupId>
      <artifactId>mysql-connector-j</artifactId>
      <scope>runtime</scope>
    </dependency>
    ```
  </Step>

  <Step title="Update application.properties">
    ```properties
    spring.datasource.url=jdbc:mysql://localhost:3306/crmdb
    spring.datasource.username=crmuser
    spring.datasource.password=your_password
    spring.jpa.database-platform=org.hibernate.dialect.MySQLDialect
    spring.jpa.hibernate.ddl-auto=validate
    ```
  </Step>
</Steps>

<Note>
  For production, use `ddl-auto=validate` to prevent automatic schema changes. Use database migration tools like Flyway or Liquibase for schema management.
</Note>

### Server Configuration

#### Port Configuration

Change the default port (8080):

```properties
server.port=8081
```

#### Context Path

Add a context path (e.g., `/crm`):

```properties
server.servlet.context-path=/crm
```

API endpoints would then be accessed at `http://localhost:8080/crm/api/user/getUsers`

### Logging Configuration

#### Enable Spring Debug Logging

Uncomment in `application.properties`:

```properties
logging.level.org.springframework=debug
```

#### Enable SQL Logging

View all SQL queries executed by Hibernate:

```properties
spring.jpa.show-sql=true
```

For formatted SQL with parameter values:

```properties
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
logging.level.org.hibernate.type.descriptor.sql.BasicBinder=TRACE
```

#### Custom Logging Levels

```properties
# Root logging level
logging.level.root=INFO

# Application logging
logging.level.com.CRM.CRM=DEBUG

# Hibernate logging
logging.level.org.hibernate=INFO
logging.level.org.hibernate.SQL=DEBUG
```

### Actuator Configuration

The application has Spring Boot Actuator enabled:

```properties
management.endpoints.web.exposure.include=*
```

This exposes all actuator endpoints at `http://localhost:8080/actuator`

**Available endpoints:**
- `/actuator/health` - Application health status
- `/actuator/info` - Application information
- `/actuator/metrics` - Application metrics
- `/actuator/env` - Environment properties

<Warning>
  Exposing all actuator endpoints (`*`) is suitable for development only. In production, limit exposure:
  
  ```properties
  management.endpoints.web.exposure.include=health,info,metrics
  ```
</Warning>

### CORS Configuration

CORS (Cross-Origin Resource Sharing) is configured using the `@CrossOrigin()` annotation on controllers.

#### Current CORS Setup

From `UserController.java` (and other controllers):

```java
@CrossOrigin()
@PostMapping("/createUser")
public ResponseEntity<User> registerUser(@Valid @RequestBody User user) {
    // ...
}
```

The empty `@CrossOrigin()` annotation allows:
- **All origins** (`*`)
- **All methods** (GET, POST, PUT, DELETE, etc.)
- **All headers**

<Warning>
  The current CORS configuration allows all origins, which is insecure for production. Restrict origins to your frontend domain.
</Warning>

#### Restrict CORS to Specific Origin

Update controller annotations:

```java
@CrossOrigin(origins = "http://localhost:3000")
```

Or for production:

```java
@CrossOrigin(origins = "https://crm.yourdomain.com")
```

#### Global CORS Configuration

For application-wide CORS settings, create a configuration class:

```java
package com.CRM.CRM.Config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.CorsRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebConfig {

    @Bean
    public WebMvcConfigurer corsConfigurer() {
        return new WebMvcConfigurer() {
            @Override
            public void addCorsMappings(CorsRegistry registry) {
                registry.addMapping("/api/**")
                    .allowedOrigins("http://localhost:3000")
                    .allowedMethods("GET", "POST", "PUT", "DELETE")
                    .allowedHeaders("*")
                    .allowCredentials(true)
                    .maxAge(3600);
            }
        };
    }
}
```

#### Production CORS Configuration

```properties
# application.properties (Spring Boot 2.4+)
cors.allowed-origins=https://crm.yourdomain.com,https://www.crm.yourdomain.com
cors.allowed-methods=GET,POST,PUT,DELETE
cors.allowed-headers=*
cors.max-age=3600
```

## Frontend Configuration

### API Base URL

The frontend currently has hardcoded API URLs in:
- `src/Routes/Home.js`
- `src/Routes/Register.js`

**Current configuration:**
```javascript
fetch("http://localhost:8080/api/user/searchUser", requestOptions)
```

### Environment Variables (Recommended)

Create React App supports environment variables. To make API URLs configurable:

<Steps>
  <Step title="Create .env file">
    Create `.env` in the frontend root:
    
    ```bash .env
    REACT_APP_API_BASE_URL=http://localhost:8080
    ```
  </Step>

  <Step title="Update API calls in source code">
    Replace hardcoded URLs:
    
    ```javascript
    // Before
    fetch("http://localhost:8080/api/user/searchUser", requestOptions)
    
    // After
    const API_BASE_URL = process.env.REACT_APP_API_BASE_URL || 'http://localhost:8080';
    fetch(`${API_BASE_URL}/api/user/searchUser`, requestOptions)
    ```
  </Step>

  <Step title="Create environment-specific files">
    - `.env` - Default for all environments
    - `.env.development` - Development environment
    - `.env.production` - Production builds
    
    ```bash .env.production
    REACT_APP_API_BASE_URL=https://api.crm.yourdomain.com
    ```
  </Step>
</Steps>

<Note>
  Environment variables must start with `REACT_APP_` to be accessible in the application. Changes to `.env` files require restarting the development server.
</Note>

### Proxy Configuration

To avoid CORS issues in development, configure a proxy in `package.json`:

```json
{
  "name": "frontend-react",
  "version": "0.1.0",
  "proxy": "http://localhost:8080",
  "dependencies": {
    ...
  }
}
```

Then update API calls to use relative URLs:

```javascript
// With proxy configured
fetch("/api/user/searchUser", requestOptions)
```

## Security Best Practices

<Accordion title="Database credentials">
  Use environment variables for sensitive data:
  
  ```properties
  spring.datasource.username=${DB_USERNAME}
  spring.datasource.password=${DB_PASSWORD}
  ```
  
  Set environment variables before running:
  ```bash
  export DB_USERNAME=crmuser
  export DB_PASSWORD=securepassword
  ./mvnw spring-boot:run
  ```
</Accordion>

<Accordion title="CORS restrictions">
  Never use `@CrossOrigin()` without parameters in production. Always specify allowed origins:
  
  ```java
  @CrossOrigin(origins = "https://crm.yourdomain.com")
  ```
</Accordion>

<Accordion title="Actuator endpoints">
  Limit actuator exposure in production:
  
  ```properties
  management.endpoints.web.exposure.include=health,info
  management.endpoint.health.show-details=when-authorized
  ```
</Accordion>

<Accordion title="SQL injection prevention">
  The application uses JPA/Hibernate, which provides protection against SQL injection through parameterized queries. Always use repository methods or JPQL queries, never concatenate user input into SQL strings.
</Accordion>

## Configuration Priority

Spring Boot loads configuration in this order (later sources override earlier ones):

1. Default properties
2. `application.properties` / `application.yml`
3. Profile-specific files (`application-{profile}.properties`)
4. Environment variables
5. Command-line arguments

**Example override:**
```bash
# Override database URL via environment variable
export SPRING_DATASOURCE_URL=jdbc:postgresql://localhost:5432/newdb
./mvnw spring-boot:run

# Override via command-line argument
./mvnw spring-boot:run --spring.datasource.url=jdbc:postgresql://localhost:5432/newdb
```
